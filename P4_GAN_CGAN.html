Informacin de la prctica

Las redes generativas adversarias (GANs) nos permiten generar datos como imgenes, vdeos o audio a partir de una entrada aleatoria. Normalmente, esta entrada se muestrea de una distribucin normal antes de pasar por una serie de transformaciones que la convierten en algo plausible, como una imagen, un sonido o un clip de vdeo.

Sin embargo, una DCGAN sencilla no nos permite controlar la apariencia (por ejemplo, la clase) de las muestras generadas. Por ejemplo, si una GAN genera dgitos manuscritos, no podramos elegir qu nmero queremos que aparezca: los resultados seran aleatorios.

Para poder controlar lo que se genera, necesitamos condicionar la salida de la GAN a una entrada semntica, como la clase de una imagen.

# Entrada: imagen de 28x28 y 1 canal, ms los mapas de clases concatenados (num_classes).
discriminator = keras.Sequential(
    [
        # Entrada del discriminador con 28x28 pxeles y canales = imagen + clases
        keras.layers.InputLayer((28, 28, discriminator_in_channels)),

        # Primera capa convolucional con activacin LeakyReLU
        layers.Conv2D(64, (3, 3), strides=(2, 2), padding="same"),
        layers.LeakyReLU(negative_slope=0.2),

        # Segunda capa convolucional con normalizacin y activacin
        layers.Conv2D(128, (3, 3), strides=(2, 2), padding="same"),
        layers.BatchNormalization(),
        layers.LeakyReLU(negative_slope=0.2),

        # Tercera capa adicional con ms filtros y regularizacin
        layers.Conv2D(256, (3, 3), strides=(2, 2), padding="same"),
        layers.BatchNormalization(),
        layers.LeakyReLU(negative_slope=0.2),
        layers.Dropout(0.2),

        # Pooling global y salida final binaria (real/falso)
        layers.GlobalMaxPooling2D(),
        layers.Dense(1),
    ],
    name="discriminator",
)

        layers.Conv2D(1, (7, 7), padding="same", activation="sigmoid"),

5. Implementacin de la clase ConditionalGAN

En esta seccin se define una clase personalizada que extiende keras.Model, y que permite controlar manualmente el proceso de entrenamiento de una GAN condicional.

A diferencia de los modelos estndar, una GAN requiere dos redes entrenadas de forma alternada:

  • El discriminador, que intenta distinguir entre imgenes reales y generadas.
  • El generador, que intenta crear imgenes lo suficientemente realistas como para engaar al discriminador.

El mtodo train_step est estructurado por bloques numerados (apartados) para entender cada etapa del entrenamiento, desde la preparacin de datos hasta la actualizacin de pesos.

Una GAN condicional aade una capa de complejidad: se entrena con imgenes y tambin con etiquetas que indican la clase deseada.

In[28]:
class ConditionalGAN(keras.Model):
    def __init__(self, discriminator, generator, latent_dim):
        super().__init__()
        # Asignar discriminador y generador
        self.discriminator = discriminator
        self.generator = generator

        # Dimensin del vector de entrada aleatorio (ruido)
        self.latent_dim = latent_dim

        # Generador de semillas para reproducibilidad
        self.seed_generator = keras.random.SeedGenerator(1337)

        # Mtricas para seguimiento de las prdidas del generador y del discriminador
        self.gen_loss_tracker = keras.metrics.Mean(name="generator_loss")
        self.disc_loss_tracker = keras.metrics.Mean(name="discriminator_loss")

    @property
    def metrics(self):
        # Lista de mtricas que se reportarn durante el entrenamiento
        return [self.gen_loss_tracker, self.disc_loss_tracker]

    def compile(self, d_optimizer, g_optimizer, loss_fn):
        # Configurar optimizadores y funcin de prdida
        super().compile()
        self.d_optimizer = d_optimizer
        self.g_optimizer = g_optimizer
        self.loss_fn = loss_fn

    def train_step(self, data):
        # Desempaquetar los datos reales (imgenes y etiquetas one-hot)
        real_images, one_hot_labels = data

        # apartado 1: Expandir y repetir etiquetas
        image_one_hot_labels = one_hot_labels[:, :, None, None]
        image_one_hot_labels = ops.repeat(
            image_one_hot_labels, repeats=[image_size * image_size]
        )
        image_one_hot_labels = ops.reshape(
            image_one_hot_labels, (-1, image_size, image_size, num_classes)
        )

        # apartado 2: Generar vectores latentes aleatorios y concatenarlos con etiquetas
        batch_size = ops.shape(real_images)[0]
        random_latent_vectors = keras.random.normal(
            shape=(batch_size, self.latent_dim), seed=self.seed_generator
        )
        random_vector_labels = ops.concatenate(
            [random_latent_vectors, one_hot_labels], axis=1
        )

        # apartado 3: Generar imgenes falsas a partir del generador
        generated_images = self.generator(random_vector_labels)

        # apartado 4: Concatenar imgenes falsas con sus etiquetas
        fake_image_and_labels = ops.concatenate(
            [generated_images, image_one_hot_labels], -1
        )

        # apartado 5: Concatenar imgenes reales con sus etiquetas
        real_image_and_labels = ops.concatenate([real_images, image_one_hot_labels], -1)

        # apartado 6: Unir el batch de imgenes reales y falsas
        combined_images = ops.concatenate(
            [fake_image_and_labels, real_image_and_labels], axis=0
        )

        # apartado 7: Crear etiquetas: 1 para falsas, 0 para reales
        labels = ops.concatenate(
            [ops.ones((batch_size, 1)), ops.zeros((batch_size, 1))], axis=0
        )

        # apartado 8: Entrenamiento del discriminador
        with tf.GradientTape() as tape:
            predictions = self.discriminator(combined_images)
            d_loss = self.loss_fn(labels, predictions)
        grads = tape.gradient(d_loss, self.discriminator.trainable_weights)
        self.d_optimizer.apply_gradients(
            zip(grads, self.discriminator.trainable_weights)
        )

        # apartado 9: Generar nuevo batch de ruido + etiquetas
        random_latent_vectors = keras.random.normal(
            shape=(batch_size, self.latent_dim), seed=self.seed_generator
        )
        random_vector_labels = ops.concatenate(
            [random_latent_vectors, one_hot_labels], axis=1
        )

        # apartado 10: Etiquetas falsas
        misleading_labels = ops.zeros((batch_size, 1))

        # apartado 11: Entrenamiento del generador
        with tf.GradientTape() as tape:
            fake_images = self.generator(random_vector_labels)
            fake_image_and_labels = ops.concatenate(
                [fake_images, image_one_hot_labels], -1
            )
            predictions = self.discriminator(fake_image_and_labels)
            g_loss = self.loss_fn(misleading_labels, predictions)
        grads = tape.gradient(g_loss, self.generator.trainable_weights)
        self.g_optimizer.apply_gradients(zip(grads, self.generator.trainable_weights))

        # apartado 12: Actualizar mtricas de prdida
        self.gen_loss_tracker.update_state(g_loss)
        self.disc_loss_tracker.update_state(d_loss)

        # Devolver las mtricas
        return {
            "g_loss": self.gen_loss_tracker.result(),
            "d_loss": self.disc_loss_tracker.result(),
        }

Si la prdida es muy alta, el generador sabr que no ha "conseguido engaar" al discriminador.

Pregunta 6.1 [0,5 pts.]: Qu pasara si se usara BinaryCrossentropy(from_logits=False) en lugar de from_logits=True? Cmo afectara eso al entrenamiento?
Respuesta a la pregunta 6.1:

Si se usara BinaryCrossentropy(from_logits=False), la funcin de prdida creera que lo que recibe ya son probabilidades entre 0 y 1, pero realmente estara recibiendo cualquier valor, ya que el discriminador no incluye ninguna funcin de activacin. Esto provocara que el entrenamiento colapsase.

Pregunta 6.2 [0,5 pts.]: Qu efecto tendra reducir o aumentar la tasa de aprendizaje (learning_rate) de los optimizadores Adam durante el entrenamiento? Por qu podra ser til usar tasas distintas para el generador y el discriminador?
Respuesta a la pregunta 6.2:

Al utilzar LR ms alta, el entrenamiento es ms rpido, pero al mismo tiempo puede provocar que el entrenamiento sea inestable. Al utilizar LR ms bajo, el entrenamiento es ms lento y ms estable. Utilizar tasas de aprendizaje distintas podra ser til dependiendo de la velocidad de aprendizaje del generador y del discriminador. Si uno de los dos aprende de base ms rpido que el otro, se podra ajustar la LR para que vayan al mismo ritmo y el entrenamiento sea ms equilibrado.

Respuesta a la pregunta 6.3:

Una posible opcin sera modificar los LR haciendo que el generador aprenda ms rpido que el discriminador. Otra opcion podra ser aumentar Dropout del discriminador para "dificultar" su aprendizaje. Otra opcin podra ser congelar el aprendizaje del discriminador en algunos batches.

En general, dificultar el aprendizaje del discriminador o agilizar el del generador.

# Funcin que genera imgenes interpoladas entre dos clases
def interpolate_class(first_number, second_number):
    # Convertimos las clases inicial y final a codificacin one-hot
    first_label = keras.utils.to_categorical([first_number], num_classes)
    second_label = keras.utils.to_categorical([second_number], num_classes)
    first_label = ops.cast(first_label, "float32")
    second_label = ops.cast(second_label, "float32")

    # Interpolamos entre las dos etiquetas (lnea recta en el espacio de clases)
    percent_second_label = ops.linspace(0, 1, num_interpolation)[:, None]
    percent_second_label = ops.cast(percent_second_label, "float32")
    interpolation_labels = (
        first_label * (1 - percent_second_label) + second_label * percent_second_label
    )

    # Concatenamos ruido + etiquetas interpoladas y generamos imgenes
    noise_and_labels = ops.concatenate([interpolation_noise, interpolation_labels], 1)
    fake = trained_gen.predict(noise_and_labels)
    return fake

# Especificamos las clases entre las que queremos interpolar
start_class = 1  # Clase inicial
end_class = 9    # Clase final

# Generamos las imgenes falsas correspondientes a la interpolacin
