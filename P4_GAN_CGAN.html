Información de la práctica

Las redes generativas adversarias (GANs) nos permiten generar datos como imágenes, vídeos o audio a partir de una entrada aleatoria. Normalmente, esta entrada se muestrea de una distribución normal antes de pasar por una serie de transformaciones que la convierten en algo plausible, como una imagen, un sonido o un clip de vídeo.

Sin embargo, una DCGAN sencilla no nos permite controlar la apariencia (por ejemplo, la clase) de las muestras generadas. Por ejemplo, si una GAN genera dígitos manuscritos, no podríamos elegir qué número queremos que aparezca: los resultados serían aleatorios.

Para poder controlar lo que se genera, necesitamos condicionar la salida de la GAN a una entrada semántica, como la clase de una imagen.

En esta práctica (PEC), construiremos una GAN Condicional que sea capaz de generar dígitos manuscritos condicionados a una clase concreta.

Instrucciones para construir los modelos:¶

  • Utiliza Sequential() de keras para definir ambos modelos paso a paso, de forma clara y ordenada.
  • Discriminador (aproximadamente 5–6 capas):
    • La entrada debe tener forma (28, 28, discriminator_in_channels), es decir, la imagen original en escala de grises más la información de clase añadida como canales.
    • Incluye varias capas Conv2D con filtros progresivos para extraer características espaciales relevantes.
    • Después de algunas convoluciones, añade BatchNormalization para estabilizar el aprendizaje y LeakyReLU como función de activación.
    • Se recomienda añadir al menos una capa Dropout para introducir regularización y evitar sobreajuste.
    • Al final, utiliza una capa de pooling global para reducir el mapa a un vector y una capa Dense que devuelva un único valor (real o falso).
  • Generador (aproximadamente 6 capas):
    • La entrada debe tener forma (generator_in_channels,), es decir, el vector latente concatenado con la clase codificada.
    • Comienza con una capa Dense que convierta ese vector en un volumen inicial (por ejemplo, 7×7×canales).
    • Usa Reshape para transformar el vector plano en un mapa 2D con profundidad.
    • Aplica varias capas Conv2DTranspose (transpuestas) para ir ampliando el tamaño de la imagen de forma progresiva.
    • Tras cada transposición, utiliza BatchNormalization y LeakyReLU para mejorar la estabilidad y la calidad de la imagen.
    • Finaliza con una Conv2D con activación sigmoid para producir una imagen en escala de grises con valores en el rango [0, 1].
  • Recuerda utilizar las variables generator_in_channels y discriminator_in_channels definidas anteriormente para que las dimensiones de entrada sean correctas.

# Crear el discriminador.
# Entrada: imagen de 28x28 y 1 canal, más los mapas de clases concatenados (num_classes).
discriminator = keras.Sequential(
    [
        # Entrada del discriminador con 28x28 píxeles y canales = imagen + clases
        keras.layers.InputLayer((28, 28, discriminator_in_channels)),

        # Primera capa convolucional con activación LeakyReLU
        layers.Conv2D(64, (3, 3), strides=(2, 2), padding="same"),
        layers.LeakyReLU(negative_slope=0.2),

        # Segunda capa convolucional con normalización y activación
        layers.Conv2D(128, (3, 3), strides=(2, 2), padding="same"),
        layers.BatchNormalization(),
        layers.LeakyReLU(negative_slope=0.2),

        # Tercera capa adicional con más filtros y regularización
        layers.Conv2D(256, (3, 3), strides=(2, 2), padding="same"),
        layers.BatchNormalization(),
        layers.LeakyReLU(negative_slope=0.2),
        layers.Dropout(0.2),

        # Pooling global y salida final binaria (real/falso)
        layers.GlobalMaxPooling2D(),
        layers.Dense(1),
    ],
    name="discriminator",
)

# Crear el generador.
# Entrada: vector latente + codificación de clase (concatenados).
generator = keras.Sequential(
    [
        # Entrada del generador: vector latente + clase codificada
        layers.InputLayer((generator_in_channels,)),

        # Primera capa totalmente conectada para mapear a 7x7xC
        layers.Dense(7*7*generator_in_channels),
        layers.LeakyReLU(negative_slope=0.2),

        # Reorganizar a mapa 7x7 con canales
        layers.Reshape((7, 7, generator_in_channels)),

        # Primera capa de transposición convolucional (upsample a 14x14)
        layers.Conv2DTranspose(128,(4,4),strides=(2,2),padding='same'),
        layers.BatchNormalization(),
        layers.LeakyReLU(negative_slope=0.2),

        # Segunda capa de transposición convolucional (upsample a 28x28)
        layers.Conv2DTranspose(128,(4,4),strides=(2,2),padding='same'),
        layers.BatchNormalization(),
        layers.LeakyReLU(negative_slope=0.2),

        # Capa final: 1 canal de salida (imagen), con activación sigmoide
        layers.Conv2D(1, (7, 7), padding="same", activation="sigmoid"),
    ],
    name="generator",
)

Pregunta 4.2 [0,5 pts.]: ¿Por qué el generador termina con una capa Conv2D con activación sigmoid? ¿Qué pasaría si se utilizara una activación relu o tanh en esa capa de salida?
Respuesta a la pregunta 4.2:

Porque sigmoid asigna cualquier valor entre 0 y 1. Mientras que tanh da una salida entre -1 y 1 y ReLU da una salida lineal solo si el resultado generado es mayor que 0, en caso contrario devuelve 0. En nuestro caso, hemos empezado normaliznado las imágenes al rango 0/1. Por lo que tanh no nos serviría. Serviría si normalizamos las imágenes en el rango -1/1. En el caso de ReLU el problema es que la salida mayor que 0 puede dar cualquier valor, no está acotado, y esto generaría imágenes defectuosas.

5. Implementación de la clase ConditionalGAN¶

En esta sección se define una clase personalizada que extiende keras.Model, y que permite controlar manualmente el proceso de entrenamiento de una GAN condicional.

A diferencia de los modelos estándar, una GAN requiere dos redes entrenadas de forma alternada:

  • El discriminador, que intenta distinguir entre imágenes reales y generadas.
  • El generador, que intenta crear imágenes lo suficientemente realistas como para “engañar” al discriminador.

El método train_step está estructurado por bloques numerados (apartados) para entender cada etapa del entrenamiento, desde la preparación de datos hasta la actualización de pesos.

Una GAN condicional añade una capa de complejidad: se entrena con imágenes y también con etiquetas que indican la clase deseada.

Lee atentamente el código y responde las siguientes preguntas:

class ConditionalGAN(keras.Model):
    def __init__(self, discriminator, generator, latent_dim):
        super().__init__()
        # Asignar discriminador y generador
        self.discriminator = discriminator
        self.generator = generator

        # Dimensión del vector de entrada aleatorio (ruido)
        self.latent_dim = latent_dim

        # Generador de semillas para reproducibilidad
        self.seed_generator = keras.random.SeedGenerator(1337)

        # Métricas para seguimiento de las pérdidas del generador y del discriminador
        self.gen_loss_tracker = keras.metrics.Mean(name="generator_loss")
        self.disc_loss_tracker = keras.metrics.Mean(name="discriminator_loss")

    @property
    def metrics(self):
        # Lista de métricas que se reportarán durante el entrenamiento
        return [self.gen_loss_tracker, self.disc_loss_tracker]

    def compile(self, d_optimizer, g_optimizer, loss_fn):
        # Configurar optimizadores y función de pérdida
        super().compile()
        self.d_optimizer = d_optimizer
        self.g_optimizer = g_optimizer
        self.loss_fn = loss_fn

    def train_step(self, data):
        # Desempaquetar los datos reales (imágenes y etiquetas one-hot)
        real_images, one_hot_labels = data

        # apartado 1: Expandir y repetir etiquetas
        image_one_hot_labels = one_hot_labels[:, :, None, None]
        image_one_hot_labels = ops.repeat(
            image_one_hot_labels, repeats=[image_size * image_size]
        )
        image_one_hot_labels = ops.reshape(
            image_one_hot_labels, (-1, image_size, image_size, num_classes)
        )

        # apartado 2: Generar vectores latentes aleatorios y concatenarlos con etiquetas
        batch_size = ops.shape(real_images)[0]
        random_latent_vectors = keras.random.normal(
            shape=(batch_size, self.latent_dim), seed=self.seed_generator
        )
        random_vector_labels = ops.concatenate(
            [random_latent_vectors, one_hot_labels], axis=1
        )

        # apartado 3: Generar imágenes falsas a partir del generador
        generated_images = self.generator(random_vector_labels)

        # apartado 4: Concatenar imágenes falsas con sus etiquetas
        fake_image_and_labels = ops.concatenate(
            [generated_images, image_one_hot_labels], -1
        )

        # apartado 5: Concatenar imágenes reales con sus etiquetas
        real_image_and_labels = ops.concatenate([real_images, image_one_hot_labels], -1)

        # apartado 6: Unir el batch de imágenes reales y falsas
        combined_images = ops.concatenate(
            [fake_image_and_labels, real_image_and_labels], axis=0
        )

        # apartado 7: Crear etiquetas: 1 para falsas, 0 para reales
        labels = ops.concatenate(
            [ops.ones((batch_size, 1)), ops.zeros((batch_size, 1))], axis=0
        )

        # apartado 8: Entrenamiento del discriminador
        with tf.GradientTape() as tape:
            predictions = self.discriminator(combined_images)
            d_loss = self.loss_fn(labels, predictions)
        grads = tape.gradient(d_loss, self.discriminator.trainable_weights)
        self.d_optimizer.apply_gradients(
            zip(grads, self.discriminator.trainable_weights)
        )

        # apartado 9: Generar nuevo batch de ruido + etiquetas
        random_latent_vectors = keras.random.normal(
            shape=(batch_size, self.latent_dim), seed=self.seed_generator
        )
        random_vector_labels = ops.concatenate(
            [random_latent_vectors, one_hot_labels], axis=1
        )

        # apartado 10: Etiquetas falsas
        misleading_labels = ops.zeros((batch_size, 1))

        # apartado 11: Entrenamiento del generador
        with tf.GradientTape() as tape:
            fake_images = self.generator(random_vector_labels)
            fake_image_and_labels = ops.concatenate(
                [fake_images, image_one_hot_labels], -1
            )
            predictions = self.discriminator(fake_image_and_labels)
            g_loss = self.loss_fn(misleading_labels, predictions)
        grads = tape.gradient(g_loss, self.generator.trainable_weights)
        self.g_optimizer.apply_gradients(zip(grads, self.generator.trainable_weights))

        # apartado 12: Actualizar métricas de pérdida
        self.gen_loss_tracker.update_state(g_loss)
        self.disc_loss_tracker.update_state(d_loss)

        # Devolver las métricas
        return {
            "g_loss": self.gen_loss_tracker.result(),
            "d_loss": self.disc_loss_tracker.result(),
        }

Pregunta 5.2 [0,5 pts.]: ¿Qué pasaría si no volvieras a generar un nuevo batch de ruido en el apartado 9 y reutilizaras el mismo que usaste para entrenar al discriminador en el apartado 2?
Respuesta a la pregunta 5.2:

Que el generador generaría la imagen a partir de una fuente que ya ha usado el discriminador, por lo que es posible que el discriminador aprovechase parte de esa información. Es decir, no serían procesos independientes y esto podría perjudicar al generador. Las redes no competirían bajo las mismas condiciones.

Pregunta 5.3 [0,5 pts.]: ¿Por qué en el apartado 10 se asigna la etiqueta 0 a las imágenes falsas al entrenar el generador, si en realidad el objetivo es que el discriminador las reconozca como reales?
Respuesta a la pregunta 5.3:

Porque el generador está intentando crear una imagen que el discriminador evalúe como real. De esta forma a continuación se le pasa la imagen y la etiqueta de real al discriminador y éste evalua la probabilidad de que sea realmente real o falsa. A continuación se evalúa la pérdida. Si la pérdida es muy alta, el generador sabrá que no ha "conseguido engañar" al discriminador.

Pregunta 6.1 [0,5 pts.]: ¿Qué pasaría si se usara BinaryCrossentropy(from_logits=False) en lugar de from_logits=True? ¿Cómo afectaría eso al entrenamiento?
Respuesta a la pregunta 6.1:

Si se usara BinaryCrossentropy(from_logits=False), la función de pérdida creería que lo que recibe ya son probabilidades entre 0 y 1, pero realmente estaría recibiendo cualquier valor, ya que el discriminador no incluye ninguna función de activación. Esto provocaría que el entrenamiento colapsase.

Pregunta 6.2 [0,5 pts.]: ¿Qué efecto tendría reducir o aumentar la tasa de aprendizaje (learning_rate) de los optimizadores Adam durante el entrenamiento? ¿Por qué podría ser útil usar tasas distintas para el generador y el discriminador?
Respuesta a la pregunta 6.2:

Al utilzar LR más alta, el entrenamiento es más rápido, pero al mismo tiempo puede provocar que el entrenamiento sea inestable. Al utilizar LR más bajo, el entrenamiento es más lento y más estable. Utilizar tasas de aprendizaje distintas podría ser útil dependiendo de la velocidad de aprendizaje del generador y del discriminador. Si uno de los dos aprende de base más rápido que el otro, se podría ajustar la LR para que vayan al mismo ritmo y el entrenamiento sea más equilibrado.

Una posible opción sería modificar los LR haciendo que el generador aprenda más rápido que el discriminador. Otra opcion podría ser aumentar Dropout del discriminador para "dificultar" su aprendizaje. Otra opción podría ser congelar el aprendizaje del discriminador en algunos batches.

7. Interpolación entre clases utilizando el generador entrenado¶

En esta actividad vamos a analizar cómo el generador responde a entradas que representan clases intermedias (no estrictamente definidas como un único dígito), lo que nos permite explorar la continuidad del espacio latente y semántico aprendido.

Se realizan dos ejercicios:

  • Interpolación entre dos clases: se seleccionan dos dígitos (por ejemplo, 1 y 9) y se generan etiquetas que van desde una codificación one-hot de la primera clase hasta la segunda. Estas etiquetas interpoladas se combinan con el mismo vector de ruido para producir imágenes intermedias. El resultado es una transición visual entre ambas clases.

  • Generación controlada por clase: se genera una imagen para cada dígito (del 0 al 9) con ruido aleatorio diferente, y se observa si el generador ha aprendido a producir ejemplos coherentes para cada clase concreta.

El resultado se visualiza como:

  • Un GIF animado para la interpolación entre clases.
  • Una figura con 10 imágenes generadas en una sola fila, una por clase.

# Función que genera imágenes interpoladas entre dos clases
def interpolate_class(first_number, second_number):
    # Convertimos las clases inicial y final a codificación one-hot
    first_label = keras.utils.to_categorical([first_number], num_classes)
    second_label = keras.utils.to_categorical([second_number], num_classes)
    first_label = ops.cast(first_label, "float32")
    second_label = ops.cast(second_label, "float32")

    # Interpolamos entre las dos etiquetas (línea recta en el espacio de clases)
    percent_second_label = ops.linspace(0, 1, num_interpolation)[:, None]
    percent_second_label = ops.cast(percent_second_label, "float32")
    interpolation_labels = (
        first_label * (1 - percent_second_label) + second_label * percent_second_label
    )

    # Concatenamos ruido + etiquetas interpoladas y generamos imágenes
    noise_and_labels = ops.concatenate([interpolation_noise, interpolation_labels], 1)
    fake = trained_gen.predict(noise_and_labels)
    return fake

# Especificamos las clases entre las que queremos interpolar
start_class = 1  # Clase inicial
end_class = 9    # Clase final

# Generamos las imágenes falsas correspondientes a la interpolación
fake_images = interpolate_class(start_class, end_class)
